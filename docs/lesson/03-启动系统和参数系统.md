# 第 3 课：启动系统和参数系统

## 本课目标

- 理解 `vision_bringup` 如何启动整套系统。
- 知道参数文件如何注入不同节点。
- 能识别 launch 中的硬件依赖、包名风险和调参入口。

## 课堂内容

### 1. vision_bringup 的角色

`vision_bringup` 本身不是算法模块。它负责把相机、识别、处理、弹道、串口、TF 这些节点组合成一套系统。

### 2. common.py

`common.py` 中定义了多个可复用节点：

- `robot_state_publisher`
- `joint_state_publisher`
- `armor_processor_node`
- `solve_angle_node`
- `serial_driver_node`

也定义了：

- `use_serial`
- `use_video`
- `node_params`
- `launch_params`

### 3. vision_bringup.launch.py

这个 launch 负责：

- 根据 `launch_params.yaml` 选择相机类型。
- 创建相机节点和识别节点容器。
- 延时启动串口、processor、solve_angle。
- 加载 `node_params.yaml`。

### 4. node_params.yaml

这里是多个节点的参数集中配置：

- `/camera_node`
- `/jlcv_serial_driver`
- `/armor_detector`
- `/armor_processor`
- `/solve_angle`

重点关注：

- 相机序列号。
- 曝光、增益、帧率。
- 串口设备和波特率。
- detector 阈值和敌方颜色。
- tracker 和 EKF 参数。

### 5. launch_params.yaml

重点配置：

- 相机类型：`hik`、`galaxy`、`mv`。
- 相机到世界/云台的外参。
- 长焦/短焦两组外参。

## 实践任务

1. 列出 `vision_bringup.launch.py` 中最终加入 `LaunchDescription` 的节点或动作。
2. 在 `node_params.yaml` 中标出：

- 相机参数
- 串口参数
- detector 参数
- tracker/EKF 参数
- solve_angle 参数

3. 写出当前默认硬件配置：

- 相机类型
- 串口设备名
- 波特率
- 相机序列号

4. 查找 launch 中是否有当前仓库不存在的包名引用。

## 检查问题

- `common.py` 和 `vision_bringup.launch.py` 分工是什么？
- `node_params.yaml` 中哪些参数不能直接照搬到另一台车？
- `use_serial` 为 `False` 时，为什么需要 `joint_state_publisher`？
- launch 中的延时启动有什么作用？
- 如果 launch 引用了不存在的包，会出现什么现象？

## 课程完成条件

完成本课需要同时满足：

- 能说明 `vision_bringup` 启动了哪些主要节点。
- 能指出 `node_params.yaml` 中至少 5 类参数的归属节点。
- 能指出 `launch_params.yaml` 中相机类型和外参配置的位置。
- 能识别当前 launch 中潜在的包名或模型配置风险。
- 能写出一份“启动前检查清单”，至少包含包名、构建、模型、串口、相机参数。
