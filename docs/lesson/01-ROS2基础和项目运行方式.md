## 课堂内容

### 1. colcon 工作空间

本项目的工作空间根目录是：

```text
auto_aim_hero_old
```

`src` 下每个带 `package.xml` 的目录会被 colcon 识别为 ROS2 包。

### 2. 常用命令

查看包：

```bash
colcon list
```

构建：

```bash
colcon build --symlink-install
```

加载环境：

```bash
source install/setup.bash
```

查看 topic：

```bash
ros2 topic list
ros2 topic info /image_raw
ros2 topic echo /processor/target
ros2 topic hz /image_raw
```

查看参数：

```bash
ros2 param list
ros2 param get /armor_detector enemy_color
```

启动 launch：

```bash
ros2 launch vision_bringup vision_bringup.launch.py
```

### 3. 本项目中的 ROS2 概念

- package：例如 `armor_detector`、`solve_angle`。
- node：例如 `armor_detector`、`armor_processor`、`solve_angle`。
- topic：节点之间传数据，例如 `/detector/armors`。
- service：一次性请求，例如 `/tracker/reset`。
- parameter：运行参数，例如 `enemy_color`、`tracker.max_match_distance`。
- launch：一次启动多个节点并注入参数。
- component node：部分节点以组件方式注册，可被容器加载。

### 4. 构建前的风险识别

构建失败不一定是代码逻辑错，可能是依赖缺失：

- ROS2 发行版不匹配。
- OpenCV、Eigen、OpenVINO 未安装。
- 海康 SDK 未安装或库路径不对。
- `serial_driver` 等 ROS 依赖缺失。
- `tools` 包依赖不可用。

## 实践任务

1. 在工作空间运行：

```bash
colcon list
```

记录每个包的路径。

2. 尝试构建：

```bash
colcon build --symlink-install
```

如果失败，记录：

- 失败包名
- 第一条真正的编译错误
- 缺失的依赖或头文件

3. 构建成功后执行：

```bash
source install/setup.bash
ros2 pkg list | grep -E 'vision_bringup|armor_detector|armor_processor|solve_angle'
```
