## 课堂内容

### 1. 节点入口

`ArmorDetectorNode` 订阅：

- `/image_raw`
- `/camera_info`
- `/game_status`

发布：

- `/detector/armors`
- `/detector/marker`
- debug 图像和 debug 数据。

### 2. 图像回调

主流程是：

```text
imageCallback()
  -> detectArmors()
  -> PnPSolver::solvePnP()
  -> 填充 Armors 消息
  -> 发布 /detector/armors
```

### 3. 传统识别路线

传统路线一般包括：

- 图像预处理。
- 颜色或亮度阈值。
- 灯条筛选。
- 灯条配对。
- 数字分类。
- PnP 位姿解算。

### 4. 当前 YOLO 路线

当前代码中识别调用偏向：

```text
detector_->yolodetect(img)
```

需要注意：

- YOLO/OpenVINO 模型路径存在个人机器绝对路径风险。
- 仓库内未看到 YOLO `.xml/.bin` 模型文件。
- launch 中出现过 `yolov5.yaml` 和 `armor_detector_rensy` 相关残留。

### 5. PnP 位姿

PnP 输入：

- 装甲板 2D 图像点。
- 装甲板实际尺寸。
- 相机内参和畸变参数。

PnP 输出：

- `rvec`
- `tvec`

最终填入 `Armor.pose`。
