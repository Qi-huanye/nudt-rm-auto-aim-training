## 课堂内容

### 1. 串口节点职责

`jlcv_serial_driver` 负责：

- 从下位机接收当前云台角、比赛状态、敌方颜色、相机选择等。
- 发布 `/joint_states` 和 `/game_status`。
- 接收 `/gimbal_command`。
- 把 pitch、yaw、control_status 打包发给下位机。

### 2. 接收包

接收包头：

```text
0x5A
```

`ReceivePacket` 包含：

- 敌方颜色。
- 比赛状态。
- 前哨站状态。
- tracker reset。
- 相机状态。
- 当前 pitch。
- 当前 yaw。
- CRC。

### 3. 发送包

发送包头：

```text
0xA5
```

发送内容：

```text
pitch
yaw
control_status
CRC
```

### 4. 串口对上层模块的影响

串口接收会影响：

- `/joint_states`，进而影响 TF。
- `/game_status.camera_status`，进而影响相机选择。
- `enemy_color`，进而影响 detector。
- `reset_tracker`，进而影响 tracker。
- 前哨站状态，进而影响 detector 忽略类别。

### 5. 断线和 CRC

如果 CRC 错误，包会被丢弃并打印错误。串口异常时会尝试 reopen。

## 实践任务

1. 画出 `ReceivePacket` 和 `SendPacket` 字段布局。
2. 写出串口节点的输入 topic 和输出 topic。
3. 记录默认串口配置：

- `device_name`
- `baud_rate`
- `flow_control`
- `parity`
- `stop_bits`

4. 设计一个模拟下位机发包工具，要求能控制：

- `enemy_color`
- `camera_status`
- `curr_pitch`
- `curr_yaw`
- `reset_tracker`

5. 设计一个模拟下位机收包工具，验证 `/gimbal_command` 是否被正确打包。
