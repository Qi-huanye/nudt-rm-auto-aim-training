## 课堂内容

### 1. solve_angle 输入输出

订阅：

```text
/processor/target
```

发布：

```text
/gimbal_command
/aiming_point
```

查询：

```text
TF: target_frame -> pitch_link
```

### 2. 控制流程

核心流程：

```text
收到 Target
  -> 判断 tracking
  -> 查询当前云台姿态
  -> 估算延迟和飞行时间
  -> 预测目标位置和旋转角
  -> 选择瞄准点
  -> 弹道求解 pitch
  -> atan2 求 yaw
  -> 加 offset
  -> 发布 /gimbal_command
```

### 3. 飞行时间和延迟

飞行时间大致由距离和弹速决定，同时会叠加系统延迟。目标运动预测依赖这个时间。

如果飞行时间估计错误，常见表现是：

- 移动目标打前或打后。
- 小陀螺目标瞄准点滞后。
- 距离变化时 pitch 偏差明显。

### 4. 弹道求解

`gravity_projectile_solver` 和 `iterative_projectile_tool` 用于求解带重力影响的 pitch。

Yaw 主要由预测瞄准点的水平位置决定：

```text
yaw = atan2(final_y, final_x)
```

### 5. gimbal_command

`/gimbal_command` 使用 `geometry_msgs/msg/Vector3`：

```text
x = pitch command
y = yaw command
z = control_status
```

`control_status` 大致语义：

- `0`：没有目标或不控制。
- `1`：跟踪中但不建议开火。
- `2`：满足更强控制/开火条件。
