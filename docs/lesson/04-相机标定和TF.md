## 课堂内容

### 1. 相机输出

`hik_camera` 节点主要发布：

- `/image_raw`
- `/camera_info`

这两个 topic 必须匹配。同一帧图像应使用对应相机的内参。

### 2. 双相机切换

代码中存在长焦/短焦两个相机句柄。`/game_status.camera_status` 会影响当前使用的相机、frame id 和 camera info。

学习时要确认：

- `camera_status == 0` 对应哪一个相机。
- `camera_status != 0` 对应哪一个相机。
- 对应使用 `camera_info.yaml` 还是 `camera_info1.yaml`。

### 3. 相机内参和外参

内参来自：

```text
camera_info.yaml
camera_info1.yaml
```

外参来自：

```text
launch_params.yaml
rm_gimbal.urdf.xacro
```

内参影响 PnP 位姿，外参影响从相机坐标系到目标坐标系的转换。

### 4. TF 链路

后续模块依赖 TF：

- `armor_processor` 需要把 `Armor.pose` 转到 `target_frame`。
- `solve_angle` 需要查询当前云台姿态。

如果 TF 不通，可能表现为：

- `/detector/armors` 有数据，但 `/processor/target` 不正常。
- `solve_angle` 报 transform error。
- yaw/pitch 方向或数值明显异常。

## 实践任务

1. 启动相机后检查：

```bash
ros2 topic list
ros2 topic hz /image_raw
ros2 topic echo /camera_info --once
```

2. 检查 `/game_status` 是否能触发相机切换。
3. 对照 `camera_info.yaml` 和 `camera_info1.yaml`，记录两套内参差异。
4. 画出你理解的 TF 链路，至少包含：

- camera optical frame
- pitch link
- yaw/world frame

实际 frame 名称以 URDF 和运行时 TF 为准。
