## 课堂内容

### 1. 项目定位

本仓库是一个 ROS2 自瞄训练项目，主体代码在：

```text
auto_aim_hero_old
```

它不是一个只包含算法函数的纯 C++ 项目，而是一个由多个 ROS2 包协作的系统。

### 2. 核心包职责

- `vision_bringup`：总启动入口，负责组合节点和参数。
- `hik_camera`：海康相机节点，发布图像和相机内参。
- `armor_detector`：装甲板识别和 PnP 位姿解算。
- `armor_processor`：目标跟踪、EKF 融合和目标状态发布。
- `solve_angle`：目标预测、弹道补偿和云台控制量输出。
- `jlcv_serial_driver`：和下位机进行串口收发。
- `jlcv_interfaces`：自定义消息和服务。
- `jlcv_description`：URDF、TF 和可视化模型。

### 3. 主数据流

需要先记住这条链路：

```text
hik_camera
  -> /image_raw, /camera_info
  -> armor_detector
  -> /detector/armors
  -> armor_processor
  -> /processor/target
  -> solve_angle
  -> /gimbal_command
  -> jlcv_serial_driver
  -> 下位机
```

下位机也会反向影响上位机：

```text
下位机
  -> jlcv_serial_driver
  -> /joint_states, /game_status
  -> 相机切换、敌方颜色、tracker reset、detector 参数更新
```

### 4. 当前仓库的已知风险

学习时要特别注意：

- `armor_detector/src/detector_node.cpp` 中存在 YOLO 模型绝对路径。
- `vision_bringup.launch.py` 中出现过 `armor_detector_rensy` 引用，但当前仓库实际包是 `armor_detector`。
- 仓库内未看到 YOLO 的 OpenVINO `.xml/.bin` 模型文件。
- 相机序列号、串口设备名、标定和外参都强依赖具体硬件。

## 实践任务

1. 进入工作空间：

```bash
cd auto_aim_hero_old
```

2. 查看 ROS2 包列表：

```bash
colcon list
```

3. 手工画出项目模块图，至少包含：

- 相机
- 识别
- 跟踪
- 弹道
- 串口
- 下位机

4. 手工画出 topic 数据流图，至少标出：

- `/image_raw`
- `/camera_info`
- `/detector/armors`
- `/processor/target`
- `/gimbal_command`
- `/joint_states`
- `/game_status`

## 检查问题
